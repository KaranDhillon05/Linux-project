📋 HOW TO USE:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: Open Your Dashboard
----------------------------
Open this URL in your browser:
    http://localhost:5001


Step 2: Launch the Workload Controller
---------------------------------------
Open a terminal and run:
    cd "/Users/karandhillon/Linux Project/SysInsight"
    ./launch_workload.sh

Choose option 1 (run inside container)


Step 3: Select What You Want to Test
-------------------------------------
You'll see a menu with options:

  1. CPU Stress      → Makes CPU usage jump to 10-30%
  2. Memory Test     → Allocates 100MB of RAM
  3. Disk I/O        → Performs file operations
  4. Mixed Workload  → All three at once! (BEST FOR DEMO)
  5. Show Metrics    → Display current stats
  6. Custom          → Create your own test
  7. Exit            → Quit the tool


Step 4: Watch Your Dashboard Update
------------------------------------
Switch to your browser and watch the graphs change in REAL-TIME!

💡 TWO MONITORING OPTIONS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Option A: Background Simulator (Already Running)
-------------------------------------------------
✓ Runs automatically in container
✓ Creates constant fluctuations
✓ Good for: Screenshots, continuous monitoring
✗ Can't control when/what runs


Option B: Interactive Menu (New! You Control It)
-------------------------------------------------
✓ YOU decide what runs
✓ YOU decide when it runs
✓ Perfect timing for presentations
✓ Specific scenarios (CPU only, Memory only, etc.)
✓ Good for: LIVE DEMONSTRATIONS


🎯 QUICK COMMANDS:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

docker-compose up -d
open http://localhost:5001

# Launch workload controller
./launch_workload.sh

# Or run directly in container
docker exec -it sysinsight python3 /home/app/web/workload_menu.py

# Check container status
docker ps

# View logs
docker logs sysinsight -f
